### Why This Proves Memory Stability
The EventProcessor's only per-window state is `event_map: HashMap<DeduplicationKey, FileEvent>`. The `flush_events` method calls `drain()` which fully empties the map. Tests verify:
- After drain, subsequent batches contain zero events from previous windows
- Over 100 cycles at high throughput, total output events == total input events
- Pause state uses `Option<Instant>` (fixed 16 bytes), not growing collections
- No `Vec::push` or `HashMap::insert` survives across window boundaries

## Session 7: Feature #4 — 异常检测规则引擎与系统通知 (2026-04-08)

### Completed
- **RuleEngine** (`detector/mod.rs`): Evaluates EventBatches against all 7 rules, sorts by severity (CRITICAL first), deduplicates alerts via AlertDeduplicator, implements AnomalyDetector trait.
- **7 Anomaly Rules** (`detector/rules.rs`):
  - RULE-01: Bulk delete > 20 files → HIGH
  - RULE-02: Bulk delete > 5 files (but ≤ 20) → MEDIUM (priority merge: doesn't fire when RULE-01 fires)
  - RULE-03: Total deletion size > 100MB → HIGH
  - RULE-04: Bulk modify > 50 files → MEDIUM
  - RULE-05: Watch root directory deleted → CRITICAL
  - RULE-06: Sensitive config file (.env, .gitconfig, etc.) modified → HIGH
  - RULE-07: Non-package modifications > 30 → MEDIUM (excludes node_modules/target/__pycache__)
- **AlertDeduplicator** (`detector/dedup.rs`): Suppresses duplicate alerts for same (directory, rule) pair within 5-minute cooldown. Supports periodic purge of expired entries.
- **NotificationDispatcher** (`notifier/mod.rs`): Routes alerts by severity — CRITICAL/HIGH sent immediately, MEDIUM queued and aggregated over 2-minute window. Implements Notifier trait.
- **MediumAggregator** (`notifier/mod.rs`): Collects MEDIUM alerts, flushes after configurable window (default 2 min), supports force flush on shutdown.
- **MacOSNotifier** (`notifier/macos.rs`): Sends macOS notifications via osascript with severity-specific formatting (icons, titles, bodies). Supports aggregated summary for MEDIUM batches.
